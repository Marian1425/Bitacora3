Formularios HTML

Ideas principales.

  • Se utilizan para hacer zonas de contacto.
  • El elemento form define un formulario que recopila información que el usuario escribe.
  • El elemento input es el más importante y se puede mostrar de diferentes formas dependiendo del atributo type.
  • Con fieldset se pueden agrupar campos y con el elemento legend se define un título para el elemento fieldset.

Apuntes de la clase.

Formularios de contacto. Sirven para hacer zonas de contacto.

Elemento “form”. Define un formulario que se utiliza para recopilar información que el usuario escribe.

Elemento “input”. Es el más importante en un formulario. Puede ser mostrado de diferentes formas dependiendo del atributo “type”.

Agrupar campos con fieldset. Se usa para agrupar datos relacionados en un formulario y el elemento “legend” define un título para el elemento fieldset.

Resumen.

Los formularios nos sirven para hacer zonas de contactos, en estas se pueden recopilar la información que el usuario escribe. Para realizar una zona de contacto es importante utilizar el elemento input el cual se muestra de diferentes maneras dependiendo del atributo type, además, también se debe usar el elemento fieldset el cual nos sirve para agrupar campos y con el atributo legend podemos definir un título para fieldset.

  • Con el elemento "input" le daremos la mayor parte del estilo a nuestro formulario.
  • Mediante el CSS podemos hacer un mejor formulario ya que le podemos dar un diseño el cual nos ayude a proyectar el estilo del proyecto web en el cual estamos trabajando.

Apuntes de la clase.

Mediante el CSS le podemos dar estilos a las etiquetas anteriormente agregadas en el HTML.

Existen una serie de propiedades en el CSS las cuales nos ayudan a hacer un diseño más optimo.

Con el CSS le podemos dar un estilo a un formulario, los cual nos ayuda a que sea más atractivo para el usuario.

Gracias al elemto "input" es que le podemos dar la mayor parte del diseño a los formularios.

Resumen.

Hacer un formulario al cual le hayamos dado un estilo en CSS siempre sera mejor para un proyecto web ya que si solo hacemos un formulario con HTML este no tendra un estilo bein definido y esto puede hacer que nuestro proyecto se vea mal, además hacer un buen formulario nos ayudara a ganarnos la confianza de nuestros usuarios.

Table

Ideas principales.

  • Las tablas sirven para organizar los datos en filas y columnas.
  • Una tabla nos permite buscar con más facilidad y rapidez.
  • Son un conjunto de datos distribuidos en filas y columnas.
  • Cuentan con 7 elementos los cuales nos permiten darle forma a la tabla.

Apuntes de la clase.

Table. Las tablas en HTML nos permiten organizar los datos en filas y columnas. Antes de la creación de CSS, los elementos HTML "table" eran frecuentemente utilizados para la disposición (posicionamiento) de una página.

¿Qué es una tabla? Es un conjunto estructurado de datos distribuidos en filas y columnas (datos tabulados). Permite buscar con rapidez y facilidad.

Elementos.

  1. thead Encabezado de la tabla.
  2. th Celda del encabezado de la tabla.
  3. tbody Cuerpo de la tabla.
  4. tr Renglón de la tabla.
  5. td Celdas de datos.
  6. caption Usado para insertar captions.
  7. tfoot Footer de la tabla.

Rowspan. Indica el número de filas que ocupara la celda. Por defecto ocupa una sola fila.

Colspan. Indica el número de columnas que ocupara la celda. Por defecto ocupa una sola columna.

Especificidad CSS

Ideas principales.

  • Sirve apara que los navegadores decidan que valores de CSS son más relevantes.
  • Se aplica cuando el mismo elemento es el objetivo de varias declaraciones.
  • Cuando varias declaraciones tienen la misma especificidad, la ultima declaración encontrada es la que se aplica.
  • Hay una jerarquía de especificidad.
  • Hay cuatro categorías que definen el nivel de especificidad.
  • Si la misma regla se escribe dos veces, siempre se aplicará la última regla.
  • La regla !important sirve para aplicar una declaración con una mayor especificidad.
  • La regla !important es una mala practica y se debe evitar.

Apuntes de la clase.

Especificidad CSS. Es el medio por el cual los navegadores deciden que valores de CSS son los mas relevantes apara un elemento y, por lo tanto, se aplicara.

Solo se aplica cuando el mismo elemento es el objetivo de varias declaraciones. Los elementos dirigidos directamente siempre tendrán propiedades sobre las reglas que un elemento hereda de su antepasado. Cuando varias declaraciones tienen la misma especificidad, la última declaración encontrada en CSS se aplica al elemento.

Jerarquía. Cada selector tiene su lugar en la jerarquía de especificidad. Hay cuatro categorías que definen el nivel de especificidad de un selector.

  1. Estilos inline. Un estilo en línea se adjunta directamente al elemento.
  2. Id’s. Es un identificador único para los elementos de la página.
  3. Clases, atributos y las pseudo-clases
  4. Elementos y seudo-elementos. Incluye nombre de los elementos y pseudoelementos.

Reglas. La última regla cuenta: si la misma regla se escribe dos veces en la hoja de estilo externa, entonces la regla inferior en la hoja de estilo esta mas cerca del elemento que se va a diseñar y, por lo tanto, se aplicara: siempre se aplica la última regla.

Selector universal. Selector universal y los valores heredados tiene una especificidad de 0-*.

La excepción !important. Cuando se usa una regla !important en una declaración de estilo, esta declaración anula cualquier otra declaración. No tiene nada que ver con la especificidad, interactúa directamente con ella. Usar !important es una mala práctica y debe evitarse porque dificulta la depuración al romper la cascada natural en sus hojas de estilo. Cuando se aplican dos declaraciones en conflicto con la regla !important al mismo elemento, se aplicará la declaración con una mayor especificidad.

Resumen.

La especificidad en CSS sirve para que los navegadores decidan que valores son los mas relevantes para un elemento y aplicarlo. Si varias declaraciones tienen la misma especificidad, la última declaración será la que se aplique al elemento. La especificidad cuenta con su propia jerarquía la cual tiene cuatro niveles. Si la misma regla se escribe dos veces, siempre se aplicará la última regla.

After y Before

Ideas principales.

  • Nos permiten insertar contenido en una página web desde CSS.
  • No es necesario tener que estar en el HTML.
  • El resultado no esta realmente en el documento, pero aparece como si lo estuviera.
  • Se debe usar con la propiedad content.
  • El content:”” sirve para insertar formas y crear figuras.
  • After sirve para insertar algo después de un contenido.
  • Before sirve para insertar algo antes del contenido.

Apuntes de la clase.

Pseudo-elementos after y before. Permiten insertar contenido en una página web desde CSS (sin tener que estar en HTML). El resultado final no está realmente en el documento, aparcere en la página como si lo estuviera. Se debe usar la propiedad content.

Content. Los caracteres especiales deben estar especialmente codificados como una entidad Unicode.

La imagen debe ser insertada en sus dimensiones exactas y no se puede cambiar de tamaño.

Content:”” Útil para insertar formas y lograr crear figuras.

After. Inserta algo después del contenido de cada uno de los elementos seleccionados.

Before. Inserta algo antes del contenido seleccionado.

Resumen.

Los pseudo-elementos aftery before nos permiten insertar contenido en una pagina web desde el CSS, para utilizar este pseudo-elemento se debe usar la propiedad content. SI deseamos insertar una imagen sus dimensiones deben ser exactas y no se puede modificar el tamaño. After y Before nos sirven para insertar algo antes y/o después de algún contenido.

Colores

Ideas principales.

  • Hay 4 formas de represenatr los colores, aunque actualmente existen mas de 7.
  • Se pueden agregar colores mediante su nombre, con caracteres, etc.
  • Cuando a un color se le agrega transparencia se le agrega el canal alfa.
  • Es muy importante seleccionar una paleta de color correcta para que la pagina sea atractiva para el usuario.
  • Los colores ayudan a que los visitantes se sientan comodos.
  • Los colores junto con el contenido determina si nos quedamos y nos vamos.
  • La pagina tiene que atraer o convencer a la vista.
  • Los colores estimulan nuestro cerebro y nos provocan emociones y sensaciones.
  • No se debe de abusar de los colores.

Apuntes de la clase.

Colores en CSS3. sRGB (Standard Red, Green, Blue), cada uno tiene 256 valores (0-255), el cero tambien cuenta. 256x256x256= 16,777,216.

Hay 4 formas de representar colores (actualmente ya hay 7 formas):

  • Keywords(palabras clave).
  • Hexadecimales.
  • RGB.
  • HSL.
  • HWB.

Keywords (Palabras clave). Son los nombres de los colores.

Hexadecimal. Trabaja con 16 caracteres 0123456789ABCDEF. El 0 representa que no hay color y el F representa el tono en su maxima intensidad. Son 7 caracteres contando el #, por ejemplo, #00FF00, los primeros dos caracteres representan el color rojo, los siguientes representan al verde y los dos ultimos representan al azul.

RGB y RGBA. Cuando queremos hacer el color con trnsparencia se agrega el canal alfa. Red, Green, Blue, Alfa.

HSL y HSLA. Hue(Matiz), Saturation(Saturacion), Lightness(Luminosidad). El Hue es un circulo y se representa en grados, Saturation y Lightness se representan el porcentaje.

HSL.

  • Hue 0-359.
  • Saturation 0%-100%
  • Lightness 0%-100%

Color y pagina web. Cuando se crea un sitio web, la eleccion de los colores es una tarea delicada, a pesar de parecer facil. Seleccionar una paleta de colores correcta es importante para comunicar eficazmente el mensaje.

Los colores en un sitio web pueden ser una herramienta muy eficaz para lograr que los visitantes se sientan comodos e interactuen con el sitio web.

Los colores junto al contenido y los otros elementos visuales son los que determinan si nos quedamos o nos vamos. Aunque no hay reglas absolutas, tener en cuenta algunas pautas pueden ayudarnbos a encontrar un diseno armonico.

Identifica el publico objetivo de tu pagina web, la mayoria de los colores estan asociados a determinados grupos. La pagina tiene ante todo que atraer y convencer a la visita. Espor eso que hay que seleccionar colores con los que la visita se sienta identificada.

Los colores estimulan nuestro cerebro y nos evocan muchas emociones y sencaiones como tranquilidad, seguridad, confianza, pasion, misterio, etc.

La rmonia de colores y el equilibrio hace que un proyecto sea mas o menos efectivo. Por lo general, se recomiendan respetar una de estas tres reglas:

  • Colores de la misma gama cromatica: es decir, diferentes tonos de un mismo color.
  • Colores complementarios: aquellos situados proximos en la gama cromatica.
  • Colores opuestos: son aquellos situados justo en el lado opuesto de la gama cromatica y que ayudan al contraste.

Otra de las recomendaciones es no abusar de los colores, dos o tres es suficeinte y siempre es bueno que haya un color predominante.

El contraste de color se produce al combinar dos colores que no estan relacionados.

Se debe eleguir una paleta de colores y usarla en todo el sitio de esta manera se le da uniformidad y sentido de unidad al sitio web. No se debe usar demasiados colores diferentes, con dos otres es suficeinte. Todo el resto deberia estar en la gama de estos colores principales.

Resumen.

Existen diferentes maneras de representar colores en el CSS, se pueden agregar mediante su nombre, caracteres, etc, cuando se le quiere agregar transparencia a los colores se agrega el canal alfa. La paleta de colores es un elemento muy importaante para hacer una pagina web ya que hace que la pagina sea atractiva para el usuario, ademas los colores ayudan a que los visitantes se sientan comodos, el color tambien es un factor importante para determinar si un usuario se queda en el sitio web o se va, pero no se deben abusar de los colores.

Header

Ideas principales.

  • Es una parte muy importante de la pagina web.
  • Es lo primero que el usuario visualiza al entrar a la pagina web.
  • Debe ser ordenada y limpia.
  • Se debe hacer un listado con los elementos que se va a incluir en el header.
  • Los elementos que se incluyen en el header depende del tipo de pagina que se realizara.
  • El logotipo es una parte muy importante de una pagina web.
  • El logotipo siempre ira en el header y suele enlazar con la página de inicio.
  • Existen varios elementos los cuales son opcionales poner en el header, por ejemplo, el menú, un buscador, los datos de contacto, un selector de idioma, entre otros.

Apuntes de la clase.

El Header. Es una de las partes mas importantes de la pagina web, pues es el elemento que los usuarioes visualizan en primer lugar cuando llegan al proyecto. La cabecera de la web debe ser ordenada y limpia para que pueda ser visualizada con un solo vistazo y no de lugar a confusion.

Tenemos que tenr en cuenta que el header es un espacio generico que tendra el mismo diseno en todas las pagina de un sitio web. Lo primero que debes hacer es un listado con los elementos que vas a incluir en el header de tu web de forma obligatoria, ya sea por motivos de branding, navegacion u otros.

¿Que elementos incluir en el header?. Van a depender de los objeivos y del tipo de web. Logotipo, menu, buscador, datos de contacto, redes sociales, carrito de compras, etc.

Logotipo. En una web no puede faltar el logotipo, sirve com o guia visual para el usuario y permite reforzar la imagen de la marca. Suele enlazar con la pagina de inicio. Es decir, permite que el usuario pueda volver al principio en todo momento. El logotipo es obligatio que ester en el header.

Elementos oipcionales que se pueden agregar en el header.

  • Nav (menu). Permite al usuario la navegacion entre las distnoitas secciones. Puede ser tambien agregado en el header. Se puede colocar un solo menu o dos, siempre en funcion del proyecto y de su estructura.
  • Buscador. Es por cuestiones de usabilidad, ya que tu conoces muy bien tu web, pero para un usuario que entra por primera vez es totalmente desconocida.
  • Datos de contacto. Se pueden incluir en el header siempre y cuando sean pocas, por ejemplo, un numero de telefono, una red social, un email, etc.
  • Tienda online. Si es una tienda onlines, se debe incluir en el header los elementos referentes al proceso de compra: carrito, cuenta, pedido, lista de deseos, inicio de sesion, etc.
  • Botones destacados. Se pueden agregar los botones destacados para enfatizar un servicio, llevar al usuario a una landing o animarle a que haga una accion especifica.
  • Barras fijas. Son muy utiles para mostrar en el header de tu erb anuncios, descuentos, cupones, llamadas a la accion o informacion adicional. Si la usas, ten cuidado de no cargarlas de informacion para no restar protagonismo al resto de elementos.
  • Selector de idioma / pais. Si el proyecto web es un sitio multi lenguaje un elemento obligatorio en el header es un selector de idioma.
  • Formulario de suscripcion. Puede ser incluido en el header, aunque no es recomendable por el gran espacio que ocupa. Los formularios de suscripcion se pueden colocar en otras partes de la web como el footer.

Resumen.

El header es una de las partes mas importantes de una pagina web ya que es lo primero que el usuario visualiza al entrar en la página, por este motivo, la barra de navegación debe ser ordenada y limpia. Los elementos que irán en el header dependerán del tipo de pagina web que se hará. El logotipo siempre debe ir en el header y usualmente siempre está situado del lado derecho, además usualmente suele enlazar con la pagina de inicio. Existen mas elementos que pueden ir en el header, pero estos elementos son opcionales.

Footer

Ideas principales.

  • Tambien se le conoce como el pie de pagina.
  • Se suele olvidar en los proyectos web.
  • Cuando esta bien diseñado puede ser muy util y efectivo.
  • Suele tener el mismo deseno en todas las paginas del sitio web.
  • Los textos legales deben estar incluidos en el footer.
  • Cookies tambien deben ser agregadas en los sitios web.
  • Las redes sociales tambien se pueden incluir en el footer.
  • Otros de los elemntos que se pueden ir en el footer son los creditos, los datos de contacto, sellos de confianza, un formulario de suscripcion, entre otros.

Apuntes de la clase.

El Footer. O pie de página es una sección olvidada en muchos proyectos web, pero si es bien diseñado, puede ser muy util y efectivo. Es un espacio egnerico que suele tener el mismo diseño en todas las paginas de un sitio web.

Al planificar cada uno de los elementos que se van a incluir en el footer es parte del proceso de diseño de un proyecto web. Asi se establecen los elementos que se van a mostrar en función de los objetivos y de la estrategia de marketing.

Textos legales. Son elementos obligatorios en en footer de toda web y no hay lugar para inerpretaciones. La privacidad de los usuarios que visitan la web es un tema muy serio y además es obligatoria. Una web tiene dos alternativas: tener avisos legales, ser infractor de la ley.

Los textos legales oblitogatorios son dos: la politica de privacidad y el aviso legal. Tambien hay otros complementarios, como son el limite de responsabilidad, las condiciones de uso, la politica de transparencia o calidad.

Cookies. La politica de Cookies es tambiñen uno de los textos legales obligatorios en todo sitio web.

Redes sociales. Son otros de los elementos que es recomendable colocar en el footer y no en el header. Permite una via de contacto directa entre la marca y las visitas de la web que se pueden convertir en seguidores. Se pueden anadir los botones de cada plataforma o utilizar widgets para mostrar las ultimas publicaciones.

Copyright / creditos. Otra información que es recomendable colocar en el footer, es la referente al copyright, osea los creditos del diseñador, los autores del material grafico como fotos, ilustraciones o videos, instituciones que colaborarn con el proyecto, si es el caso.

Datos de contacto. Son una información muy util para que las visitas se pongan en contacto con la empresa o negocio. Se pueden incluir datos como telefono, email, direccion incluso se puede incrustar un mapa de Google Maps.

Sellos de confianza. En webs de sectores como el tecnologico, sanitario, comercio o industrial es común encontrar sellos de confianza que avalan la actividad y profesionalidad de dicha empresa. El sello de Confianza es un distintivo que otorga la Asociación de Internet MX a sitios web que pertenecen a empresas, organizaciones, instituciones y personas identificadas y comprometidas con la generación de confianza en línea.

CTA (Call To Action). Es una imagen o un texto que invita a los visitantes de una página weba realizar alguna acción. Su traducción es “llamada a la acción” y hace referencia a cuando queremos impactar.

Sitiemap o mapa de sitio. Son archivos en los que se proporciona información sobre las paginas, los videos y otros archivos del sitio, asi como sobre las relaciones que hay en ellos.

Formularios de suscripción o Newslatter. E sun formulario ubicado en cualquier pñagina de un sitio web o blog donde los usuarios pueden completar los campos con sus datos para recibir correos electronicos sobre temas relacionados con su interes. El objetivo principal de formulario de suscripción es atraer nuevos suscriptores a tu lista de correo.

Resumen.

El footer tambien conocido como el pie de pagina, puede ser muy util cuando esta bien diseñado aunque casi siempre se suele olvidar en los proyectos web, esxisten varios elementos que pueden ir en el footer, los cuales ayudan a que el footer quede mejor diseñado, por ejemplo, los textos legales, la politica de cookies, las redes sociales, los creditos, un formulario de suscripcion, entre otros.
